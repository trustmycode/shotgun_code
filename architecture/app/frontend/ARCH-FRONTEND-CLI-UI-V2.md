## Контекст
Этот компонент представляет собой пользовательский интерфейс на Шаге 4, который позволяет пользователю взаимодействовать с CLI-инструментом. В версии 2 планируется адаптация под новый асинхронный поток авторизации `codex-cli`.

## Структура
Основной затронутый файл: `frontend/src/components/steps/Step4ApplyPatch.vue`.
Компонент будет переработан для управления несколькими состояниями:
- Проверка установки/авторизации.
- Предложение установки.
- Предложение авторизации.
- Ожидание подтверждения авторизации от пользователя.
- Готовность к выполнению.

## Поведение
При входе на Шаг 4 UI будет проверять статус `codex-cli`. Если требуется авторизация, пользователю будет предложено нажать кнопку "Войти". Это вызовет бэкенд-функцию, которая откроет новый терминал. UI перейдет в состояние ожидания, показывая сообщение и кнопку "Проверить статус входа". Пользователь должен будет завершить вход во внешнем терминале, а затем нажать эту кнопку для повторной проверки статуса.

- Реализовать новый многоэтапный UI на Шаге 4 для поддержки установки и асинхронной авторизации `codex-cli`.
